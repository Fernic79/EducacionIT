Clase 11 - Formularios

En esta clase vamos a trabajar con formularios de todo tipo.

Empezamos con un formulario sencillo

La etiqueta que usaremos sera FORM y desde ahi cargarlo con sus atributos, uno de ellos es el METHOD (hace referencia a los métodos http) en referencia a ello tendremos el metodo GET (obtener datos sin modificar la base de datos, solicita info del server x), la otra forma de metodo es POST (envia y modifica datos en la base de datos) este tambie puede llegar a servir para modificar datos en la base.

Otro atributo es el ACTION (sirve para matchearlo a un archivo externo) para ello le damos la ruta del archivo en el atributo; en ello segun el metodo que utilice todos los datos enviados iran al archivo de la ruta.

La etiqueta LABEL (nos servira para que el usuario vea lo que solicitamos) es una etiqueta que muestra informacion al usuario para que sepa que datos debe completar, por ejemplo cuando le pedimos un "nombre" en el formulario.

Otra etiqueta que utilizaremos es INPUT (esta etiqueta le da al usuario un espacio para que complete los datos solicitados) en esta etiqueta veremos un recuadro para llenar con los datos solicitados. Trae un atributo TYPE (define los tipos de adtos que pueden solicitarse) especifica que datos pueden completar en el espacio, por ejemplo "TEXT". La etiqueta INPUT en sus atributos debe contar con un ID que estará vinculado al atributo FOR del LABEL para saber que los datos que no s de el usuario vayan con ese nombre a buscar la variable del archivo del ACTION.

El BUTTON de tipo RESET nos sirve para borrar los datos ingresados, lo mismo puedo hacerlo con la etiqueta INPUT.

Por tema de validaciones de datos puedo agregar atributos que sirvan para corroborar los datos solicitados cuando los ingresa el usuario (min, max, minLenght, maxLenght, etc), más adelante lo puedo sumar a las validaciones por javascript. Y el atributo REQUIRED que obliga al usuario a cargar datos en el campo.
