Seguimos trabajando con FORM

Trabajamos el atributo ENCTYPE que posee tres formas de uso (datos codificados; datos multiplataforma; texto plano html5) según lo que trabajemos es lo que utilizaremos.

Otro atributo que podemos utilizar es el AUTOCOMPLETE es una forma de que el navegador le ayude a completar los datos que debe ingresar.

Con FIELDSET podemos darle un marco al formulario. Con la etiqueta LEGEND podemos ponerle un titulo que quedara en el fieldset. Con estilos por CSS podemos darle más forma.

Mientras el ID se toma para utilizar con el DOM, el NAME lo utilizaremos para las plantillas (esta más orientado a las variables) en el NODE. Para entender un poco más en el FRONT tendremos normalmente 3 capas (html, css y js), en el BACK tendremos distintas capas (react, node, js, etc) todo depende de lo que queramos hacer.

PLACEHOLDER es un atributo que le sugiere al usuario los datos a completar.

Para cuando utilizamos emails podemos agregarle el atributo PATTERN (un patróm de diseño para que el usuario introduzca) dentro del pattern utilizaremos arrays para ofrecer conjunto de cosas entre [], para ello la estructura que se indica para completar será la siguiente [letras y números]+@[empresa de mail]+\.[extension]{cantidad de letras nimimas y máximas 2,15}, veremos que en el ejemplo abarca todas las letras minúsculas y mayúsculas, números y símbolos. Hoy en día esto se trabaja con librerías que validan los datos que se ingresan.

El label de PASSWROD nos sirve para solicitar una clave al usuario y en la página los caracteres no se verán y en su lugar aparecerán asteriscos.

Vemos algo de GIT

Hasta ahora venimos trabajando en una solo rama la MASTER, pero el git nos permite tener tantas ramas como queramos. Para crear una rama primero debemos ver cuantas ramas tenemos (git branch) y veremos cuantas hay. Para crear rama entonces utilizaremos el mismo comando y el incluimos el nombre (git branch rama1). Sabemos en que rama estamos trabajando cuando vemos en la terminal que aparece con un * y esta pintada de color.Para moverme de una rama a la otra utilizo el comando y su nombre de rama (git checkout rama1).

De esta forma puedo trabajar con una página que voy a utilizar con cambios por algún evento y luego de dicho evento volver a la página de la master. Como git registra cada vez que yo hago modificaciones, podemos volver a un punto pasado gracias a que tengo el historial de cambios (git log me muestra el historial de cambios).

Para subir las ramas debo subirlas con git push origin y nombre de la rama. Termino la clase con comentarios de git.
